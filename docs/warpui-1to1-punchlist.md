## THE 3 BIGGEST PIECES OF WORK
1. **Recursive Node/Layout pane tree + per-tab layouts** (Center) — replaces the hard-coded `SplitRow` in `/Users/rajpootathar/ideaProjects/crane/crates/warp_term_spike/src/shell.rs:273` and generalizes `/Users/rajpootathar/ideaProjects/crane/crates/warp_term_spike/src/split.rs`. 2. **Real git data layer + the whole Changes/Files Right Panel** — there is no git module in the crate; `right_sidebar()` (shell.rs:209) and `files_pane()` (shell.rs:284) are static mocks. Needs a ported `git.rs`, a real recursive FS tree, staging, commit box, diff-on-click.

Left-panel rename, tint, reorder, attention, and right-panel refresh all require promoting this to a live mutable model with write-back.
