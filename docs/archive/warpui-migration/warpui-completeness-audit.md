# Crane warpui — completeness vs old Crane (v2 audit)

> Generated by a multi-agent audit (14 area agents reading old egui Crane vs new warpui Crane, then synthesis). Date: 2026-06-29.

## 1. Overall completeness estimate

**Roughly 20–25% of old Crane is present in the warpui port.** The terminal grid/PTY core renders and the split/tab/sidebar scaffolding exists as read-only display shells, but nearly every interaction layer — git mutation, persistence, the editor, doc panes, modals, clipboard, and most shortcuts — is absent. What exists is a navigable skeleton; what's missing is almost all of the daily-driver behavior.

## 3. Ordered P0 punch list (to reach daily-usable parity)

1. **Terminal interaction core.** Selection model + Cmd+C copy, Cmd+V bracketed paste, Cmd+A select-all, Cmd+K clear, scrollbar, URL/path detection. The agent-CLI workflow is the product's reason to exist — without copy/paste the terminal is not usable for real work.
2. **Session persistence write-back.** Debounced atomic save (tmp→rename, `.bak`), full-tree serialize, typed `Session` read (layout tree, cursor, ids), terminal ANSI-history restore. Without this every feature below is lost on restart.
3. **Git mutation layer.** `stage`/`unstage`/`commit`/`push`/`pull`/`fetch` + `current_branch`/`ahead_behind`, behind a threaded `dispatch_git_op` with status gating + `wake()`-on-completion. Harden parser to `-z` + `--untracked-files=all`.
4. **Right panel Changes — staging + commit.** Grouped collapsible tree, tri-state stage/unstage, staged/unstaged split, commit box + Cmd+Enter, branch toolbar (Fetch/Pull/Push), op-error pill, scroll, background status poll, file-click → open pane.
5. **Left panel — project/workspace management.** Add/Remove Project (rfd), per-row +worktree/+tab/×, inline rename, right-click context menus, stable `u64` row ids (replace positional `(idx,idx,idx)`).
6. **Pane layout interaction.** `focus_next`/`focus_prev` + Cmd+[/], dock-zone drag (4 edges + Center→swap) replacing hardcoded-Horizontal, wire dead `SplitPane` to header buttons, inactive-pane dim/focus-border, press-to-focus body, Esc-to-restore maximize.
7. **Keyboard shortcut completeness.** Fix Cmd+T (split, not new-tab), add Cmd+Shift+T (new tab), Cmd+Shift+W (close tab), Cmd+[/], Cmd+=/-/0 font zoom, Cmd+C/V/A/K.
8. **App-shell window basics.** window_bounds (1480×920) + min size + persistence, native menu bar, quit/close-confirm, on_exit/Drop teardown.

## 4. Wins already at / near parity

- **Terminal grid + PTY core** — VT parsing, grid render, per-pane PTY spawn, `Drop` child-kill + reader-join. Ctrl-based C0 codes work.
- **App icon** via `NSApplication::setApplicationIconImage`.
- **Repaint model** — event-driven `async_channel` + `spawn_stream_local` + `notify()`.
- **Pane maximize** — `maximized` + `ToggleMaximize` + `center()` short-circuit (missing Esc-to-restore).
- **Splitter live-resize** with ratio clamping + hover cursor.
- **Tab switch / add / close** wired (missing rename/reorder/tint/confirm).
- **Left-tree navigation** — expand/collapse + selection highlight (read-only).
- **Theme color tokens** — `crane-dark` ported verbatim (not switchable yet).
- **Multipane** — recursive Node tree, Cmd+D/Cmd+Shift+D split, Cmd+W close, drag-rearrange (horizontal), pane headers.

**Bottom line:** the port is a rendering-and-navigation skeleton. The terminal looks right and panes split, but it is not yet daily-usable: you cannot copy out of the terminal, your session evaporates on restart, and you cannot commit. Items 1–4 are the minimum bar for "I can use this instead of old Crane for a day."
